# 情绪递质模块（EMgr/NT）设计文档（模块 6）

生成时间：2026-04-12  
理论依据：`txt版本的理论核心.txt`（重点参考：3.3.1、3.9、3.10、3.12）

> EMgr（情绪管理器）在 AP 中是“慢变量调制器”：不替代结构认知与查存一体，而是把 CFS、奖惩信号与行动反馈整合为 NT（递质通道）状态，并输出调制量影响注意力、学习与行动风格。

---

## 1. 模块定位与职责边界

### 1.1 理论定位（3.9）
EMgr 的工作目标是把：
- CFS（违和、惊、期待、压力、正确事件、把握感、繁/简等）
- Rwd/Pun（奖励/惩罚信号）
- 行动反馈（成功/失败、耗时、成本、任务进展等，可选）

映射为一个可观测、可衰减、可上下限约束的通道集合 NT，并输出对其他模块的调制参数。

### 1.2 工程职责
- 维护 `nt_state`（通道字典 + 时间戳 + 衰减）
- 维护 `rwd_pun_snapshot`（奖励/惩罚底色与短期尖峰）
- 输出 `modulation`（给 AF/HDB/Drive 的调制参数包）
- 提供审计：每次更新的输入摘要、脚本版本、输出变化量

### 1.3 不做什么
- 不直接决定“选哪个结构/写哪个结构”（HDB）
- 不直接决定“执行哪个行动”（Action/Drive）
- 不把情绪当作自然语言风格生成（这里只提供参数，表达由行动器/LLM 完成）

---

## 2. 输出接口：影响哪些参数（理论 3.9.2 必须对齐）

理论明确给出 NT 对外的典型调制接口（映射由 IESM 脚本配置）：

### 2.1 影响 AF（注意力滤波器）
- 注意力预算上限（一次 Tick 可保留多少 CAM 对象）
- 截断阈值与增益曲线（对小波峰更敏感/更迟钝）
- 疲劳与探索偏置（重复路径抑制强度、探索新路径倾向）
- 聚焦/警觉模式切换阈值（带参聚焦 vs 不带参警觉）

### 2.2 影响 HDB 的学习力度
- 结构数据库权重更新系数（实能量增强系数、虚能量磨损系数）
- 感应赋能阈值与传播系数（虚传播强度、诱发强度）
- 新结构生成敏感度（共同部分切割触发严格程度）
- 闲时巩固触发门控（整理/压缩/重写索引）

### 2.3 影响 Drive 与行动系统
- 驱动力增益（更愿意行动/更不愿意行动）
- 行动阈值缩放（更冲动/更保守）
- 行动失败后的重试策略倾向（坚持/转向/求助/暂停）
- 表达风格与社交倾向（更亲和/更冷静/更防御）

## 3. 默认通道方案（对齐理论 3.9.3，并补齐工程调制状态）

理论给出的六个拟人化通道依旧保留，并继续作为默认主通道：
- DA（多巴胺/奖励驱动）
- ADR（肾上腺素/警觉唤醒）
- OXY（催产素/亲和连接）
- SER（血清素/稳定满足）
- END（内啡肽/舒缓镇痛）
- COR（皮质醇/长期警戒）

考虑到原型系统已经有“注意力收窄/发散”“新鲜感/重复感”“把握感”等系统级状态，因此默认工程实现额外补充两个功能通道：
- NOV（新颖探索）：专门承接“意外、新鲜、未证实预测”带来的探索倾向，调制近因偏置、显著性权重与虚能量扩散强度。
- FOC（专注锁定）：专门承接“高把握/高压力下的持续聚焦”倾向，调制 Top-N 收窄、focus_boost、阈值收束与传播收束。

工程口径上，通道只是“可订阅调制变量”，名称借用生理学术语降低理解门槛；核心不是拟生物，而是：
- 可配置的更新/衰减/上下限
- 与 AF/HDB/Drive 的真实参数调制挂钩
- 允许新增通道，但新增通道必须同时定义输入条件、衰减规则和调制出口

## 4. 接口设计（原型阶段建议）

### 4.1 输入
建议输入事件（每 tick 1 次）：
- `cfs_signals`（来自模块 5）
- `reward_punishment_signals`（可来自结构中的属性刺激元或外部反馈编码）
- `action_feedback`（来自模块 7 的执行结果，可选）
- `resource_metrics`（tick 用时、展开轮次等，可选，给 APT 用也可）

### 4.2 输出
输出 3 类对象：
1. `nt_state_snapshot`：通道值字典（含衰减后值、上下限）
2. `rwd_pun_snapshot`：奖励/惩罚摘要（强度、来源、绑定对象）
3. `modulation`：对 AF/HDB/Drive 的调制参数包

### 4.3 预留：脚本化更新与审计
EMgr 的更新规则应尽量放在 IESM 的 “EMgr/NT 脚本”中（3.10.1），以便：
- 版本化
- 可回滚
- 可审计
- 冲突仲裁（安全类优先）

---

## 5. 现阶段接入策略

当前实现已不再停留在“只展示、不生效”的阶段，而是采用：
- `CFS -> NT`：违和、惊、期待、压力、把握感、繁/简、重复感、验证态共同更新 NT
- `Rwd/Pun -> NT`：奖励/惩罚全局底色继续推高或抑制相关通道
- `NT -> Attention`：真实调制 `top_n`、`min_cam_items`、能量权重、显著性权重、近因偏置、focus_boost、截断曲线
- `NT -> HDB`：真实调制 `base_weight_er_gain`、`base_weight_ev_wear`、`ev_propagation_ratio`、`ev_propagation_threshold`、`er_induction_ratio`
- `NT -> Drive`：通过行动阈值缩放，让奖赏驱动、长期警戒、探索/专注倾向对执行果断性产生真实影响

后续仍建议把更多细粒度更新规则逐步外提到 IESM 脚本层，但默认运行口径已经要求：情绪递质不是标签，而是实际参与系统调参的慢变量。
