# NT 八通道详细设计与现状报告（2026-04-28）

本文面向当前默认运行口径，解释 AP 原型中情绪递质/调制通道（NT, NeuroTransmitter channels）的**真实实现现状**，重点回答：

1. 这 8 个通道分别代表什么。
2. 它们怎样上升、怎样下降。
3. 它们会影响哪些运行参数。
4. 它们长期维持在不同区间时，会把系统塑造成怎样的认知/行动风格。

---

## 1. 当前默认八通道

当前默认启用 8 个 NT 通道：

- `DA`：多巴胺（趋近 / 奖励推进）
- `ADR`：肾上腺素（警觉 / 唤醒）
- `OXY`：催产素（联结 / 亲和）
- `SER`：血清素（稳定 / 满足）
- `END`：内啡肽（缓冲 / 恢复）
- `COR`：皮质醇（警戒 / 保守）
- `NOV`：新颖探索
- `FOC`：专注锁定

其中：

- 前 6 个是原先的核心通道；
- `NOV` 与 `FOC` 是为对齐新理论口径新增的两个**功能性调制通道**；
- 它们不是“情绪标签”，而是系统的**慢变量全局调制层**。

## 2. NT 的统一更新规则

### 2.1 每个 tick 的更新顺序

当前 `EmotionManager.update_emotion_state(...)` 的流程是：

1. 先对现有通道值做一次**回归基线的衰减**。
2. 从 `CFS`（认知感受信号）计算一份 NT 增量。
3. 从 `Reward/Punish`（奖励/惩罚快照）再计算一份 NT 增量。
4. 若有先天脚本或外部脚本指定通道增量，再叠加一份脚本增量。
5. 合并所有增量后，应用到当前 NT 状态。
6. 基于更新后的 NT 状态，计算下一 tick 要生效的调制输出：
   - 注意力调制
   - HDB 学习/传播调制
   - 行动阈值调制（在 action 模块消费）

### 2.2 衰减不是回零，而是回基线

每个通道都有 `base` 基线值与 `decay_ratio`。

当前衰减公式：

`decayed = base + (value - base) * decay_ratio`

含义：

- 通道值高于基线时，会向下回落；
- 通道值低于基线时，会向上回弹；
- 所以系统长期不会把所有通道都压成 0，而是维持一组“性格底色”。

### 2.3 正向增量使用软上限

对正增量，当前不是简单线性累加，而是使用“靠近上限但不打满”的软饱和方式：

`after = max - gap * exp(-delta / k)`

这意味着：

- 通道越高，再往上抬就越难；
- 强烈事件仍然能把通道推高，但不容易长期贴死在 1.0；
- 更适合长期运行与审计。

### 2.4 负向增量目前是线性下降

当前若某通道收到负向增量：

- 直接按线性方式下降；
- 再被 `min/max` 钳制；
- 然后在下一轮继续向基线回归。

所以“下降”的来源主要有三类：

1. 负向 CFS 映射；
2. 负向 Reward/Punish 映射；
3. 自然衰减回基线。

## 3. 通道总表（默认参数）

| 通道 | 基线 base | 衰减比 decay_ratio | 软上限 k | 直观解释 |
|---|---:|---:|---:|---|
| DA | 0.12 | 0.91 | 0.35 | 趋近、奖励推进、愿意做/愿意学 |
| ADR | 0.05 | 0.85 | 0.35 | 紧急唤醒、警觉扫描 |
| OXY | 0.12 | 0.93 | 0.35 | 联结、协同、温和整合 |
| SER | 0.18 | 0.94 | 0.35 | 稳定、满足、抗波动 |
| END | 0.10 | 0.91 | 0.35 | 缓冲、止痛、恢复 |
| COR | 0.06 | 0.86 | 0.35 | 警戒、保守、防错 |
| NOV | 0.08 | 0.89 | 0.33 | 给新线索/意外路径机会 |
| FOC | 0.10 | 0.92 | 0.34 | 锁主线、收搜索、稳聚焦 |

补充理解：

- `decay_ratio` 越低，离开事件后越快回到基线；
- `base` 越高，越像系统的稳定“性格底色”；
- `soft_cap_k` 越小，越容易在高位开始饱和。

## 4. 八个通道逐个说明

## 4.1 DA（多巴胺）

### 代表什么

- 奖励推进
- 趋近意图
- “继续做 / 值得试 / 值得巩固”

### 主要上升来源

- `correct_event`
- `expectation`
- `expectation_verified`
- `reward` 全局奖励快照

### 主要下降来源

- 当前没有显式的强负向 CFS 直压 `DA`
- 主要依靠：
  - 自然衰减回 `0.12`
  - 脚本负增量（若显式提供）

### 主要影响

- 行动阈值下降（更容易行动）
- 注意力 `top_n` 轻微增加
- 注意力总能量权重略升
- 注意力近因增益上升
- HDB 的 `base_weight_er_gain` 上升
- HDB 的 `ev_propagation_ratio` 上升
- HDB 的 `ev_propagation_threshold` 下降
- HDB 的 `er_induction_ratio` 上升
- HDB 的 `base_weight_ev_wear` 下降（更容忍探索）

### 长期高位的系统风格

- 更愿意尝试与执行
- 更容易把现实成功经验快速沉积
- 更容易从现实证据继续诱发新预期
- 但若长期偏高，可能过于乐观、过快推进

- HDB 的 `ev_propagation_ratio` 上升
- HDB 的 `er_induction_ratio` 上升

### 长期高位的系统风格

- 变得更警觉、更快切换、更容易关注突发
- 对新刺激更敏感
- 但长期高位会让系统更难稳定收敛

## 5. NT 会影响哪些参数

## 5.1 注意力参数

当前 NT 会调制这些注意力输出：

- `top_n`
- `min_cam_items`
- `priority_weight_total_energy`
- `priority_weight_cp_abs`
- `priority_weight_salience`
- `priority_weight_fatigue`
- `priority_weight_recency_gain`
- `focus_boost_weight`
- `min_total_energy`
- `keep_score_ratio_base`
- `keep_score_ratio_concentration_gain`
- `keep_score_ratio_min`
- `keep_score_ratio_max`

可以粗略理解为三大类：

1. **看多宽**：如 `top_n`、`min_cam_items`
2. **按什么排序**：如总能量、认知压、显著性、疲劳、近因
3. **收得多紧**：如聚焦增益、最小能量门槛、保留比例

---

## 5.2 HDB 参数

当前 NT 会输出 HDB 的运行时缩放：

- `base_weight_er_gain_scale`
- `base_weight_ev_wear_scale`
- `ev_propagation_threshold_scale`
- `ev_propagation_ratio_scale`
- `er_induction_ratio_scale`

它们分别对应：

- 更愿不愿意把现实证据学进去
- 更愿不愿意让纯虚循环继续活着
- 虚能量扩散要不要更谨慎
- 虚能量扩散预算有多大
- 现实证据能诱发多少新预期

---

## 5.3 行动参数

当前 Action 模块主要消费 NT 的方式是：

- 把 NT 状态映射成 `threshold_scale`
- 最终作用在 `effective_threshold = base_threshold * threshold_scale`

默认方向：

- 降阈值：`DA / ADR / OXY / END / NOV / FOC`
- 升阈值：`SER / COR`

其中：

- `COR` 升阈值最强；
- `DA` 降阈值最强；
- `FOC` 与 `NOV` 是温和降阈值。

这意味着：

- 高 `COR`：更不想轻易出手；
- 高 `DA/NOV/FOC`：更容易真正把动作推过线。

## 6. 长期层面的总体效应

如果把 8 个通道看成系统的“运行风格旋钮”，那么：

- `DA + NOV + ADR` 偏高：
  - 系统更愿意试、扫、发散
  - 容易探索、行动、拓路径
  - 但可能噪声变多、主线不稳

- `FOC + SER` 偏高：
  - 系统更稳、更收束
  - 更能盯住主线和已有叙事
  - 但可能探索不足、泛化变弱

- `COR` 偏高：
  - 系统更保守、更防错
  - 对虚扩散与轻率行动更敏感
  - 长期会把系统塑造成“谨慎但僵”

- `END + OXY` 偏高：
  - 更缓冲、更整合、更不容易被局部惩罚打散
  - 长期有助于稳定恢复与温和联结

理想上并不是某一条永久最高，而是：

- 遇到新线索、意外变化时：`NOV/ADR` 短时升；
- 明确主线后：`FOC/SER` 接管；
- 风险明显时：`COR` 上升做保守门控；
- 高压后：`END` 帮助缓冲恢复；
- 奖励/正确积累时：`DA/OXY` 推进学习与行动。

## 7. 当前实现的边界

当前版本已经实现的，是：

- 8 通道状态维护
- 基线回归衰减
- CFS -> NT
- Reward/Punish -> NT
- NT -> 注意力调制
- NT -> HDB 调制
- NT -> 行动阈值调制
- 观测台图表与实验指标导出

当前还不是“最终生物学拟真”的地方：

- 各通道仍是功能性工程通道，而不是追求神经科学教科书级拟真；
- 目前重点是**让系统行为风格可解释、可调、可验证**；
- 未来仍可以继续扩展更多全局调制通道，但应坚持“有真实系统级作用再新增”。
