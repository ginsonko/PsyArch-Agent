# AP 最小原型观测台（Observatory）运行说明书

这是一份极简运行指南，用来启动本项目的本地“观测台”并输入文本观察流程日志。

重要说明（先说在前面）：
本项目仍处于原型阶段，目标以验证流程与对齐理论口径为主。代码与细节都在持续开发和完善中，可能出现各种意外行为或不符合预期的地方。如果你发现 bug 或疑问，请在仓库里提交 issue，或加入 README 中的反馈群沟通。

## 方式 A：一键运行（推荐）

1. 双击运行 `依赖自检与安装.bat`
2. 安装完成后，双击运行 `快速启动观测台.bat`
3. 默认会自动打开浏览器。如果没有自动打开，请手动访问：
   `http://127.0.0.1:8765/next/`
4. 在页面顶部输入框中输入一段短文本，然后运行一次循环观察流程。

输入建议：
为了便于观测与避免日志过长，建议先用 1 到 30 个字以内的短文本进行测试，例如 `你好`、`你好呀!`、`今天有点压力`。

## 方式 B：手动运行（PowerShell）

在项目根目录（和这些 `.bat` 文件同级）打开 PowerShell，依次执行：

```powershell
cd <你的项目目录>

python -m venv .venv
.\.venv\Scripts\activate

python -m pip install -U pip
python -m pip install -r requirements.txt

python -m observatory
```

然后在浏览器打开：
`http://127.0.0.1:8765/next/`

## 常见问题（FAQ）

1. `jieba` 没装会怎样？
系统会自动关闭 `jieba` 分词，并回退到字符级切分继续运行，不会因为缺少 `jieba` 而中断流程。

2. 为什么不建议输入太长？
当前原型以“可观测性与流程审计”为主，日志信息会比较多。输入太长会导致页面信息密度很高，不利于理解和排查问题。

3. 运行时遇到问题怎么办？
请把你输入的文本、时间点、页面截图或日志片段整理后在仓库里提交 issue，或加入 README 中的反馈群沟通。提交前请先删除 API Key、QQ 号、聊天记录等隐私信息。
