# 旧版数据迁移与原地更新

如果本地同时出现 `PsyArch-Agent` 和 `PsyArch-Agent-main`，通常是因为：

- `PsyArch-Agent-main`：从 GitHub 下载 ZIP 后解压出来的目录；
- `PsyArch-Agent`：一键拉取脚本用 Git 克隆出来的目录。

这两个目录是两份独立程序。你在哪个目录里填写 API Key、人设、QQ 白名单、日记、图书馆和 HDB 数据，这些数据就留在哪个目录里。

## 推荐方案：原地更新当前目录

如果旧目录里已经有你的配置和聊天数据，最稳妥的方式是直接更新这个旧目录本身，而不是再新建一个目录。

把下面这个文件放进你正在使用的 PA 目录里，然后双击：

```bat
一键重启进程并更新.bat
```

它会自动执行：

1. 关闭占用 `8765` 端口的旧 PA 后端进程；
2. 在当前文件夹内拉取或覆盖更新最新版 PA；
3. 无论更新成功或失败，都重新启动 PA-Agent 与 AP 后端；
4. 打开 `http://127.0.0.1:8765/next/`。

如果当前目录是 Git 仓库，它会执行 `git fetch` / `git pull`。如果当前目录是 GitHub ZIP 解压出来的目录、没有 `.git`，它会临时下载最新源码并覆盖到当前目录，同时保留：

- `observatory/outputs/` 里的配置、对话状态、日记、定时任务、图书馆、表情包、附件和生成图片；
- `hdb/data/` 里的 AP/HDB 长期运行数据；
- `.venv/`、`node_modules/`、日志、NapCat 等本地私有内容。

## 方案二：迁移旧目录数据到新版目录

如果你已经决定以后只使用新的 `PsyArch-Agent` 目录，可以在新版目录里双击：

```bat
一键迁移旧版PA用户数据.bat
```

脚本会自动尝试寻找同级的 `PsyArch-Agent-main`，也可以手动粘贴旧目录路径。迁移时会复制用户数据，并先备份新版目录中已有的数据；旧目录不会被删除。

默认迁移范围包括：

- `observatory/outputs/agent` 下的配置、人设、对话状态、日记、定时任务、图书馆、表情包、附件、生成图、运行包等非日志数据；
- `observatory/outputs/auto_tuner` 下的 AP 自动调参配置和持久化覆盖；
- `hdb/data` 下的 AP/HDB 长期运行数据。

默认不会迁移巨大的 `.jsonl` / `.log` 历史日志，避免把旧版已经膨胀的日志一起搬到新版。

## 方案三：打包迁移

如果旧目录和新目录不在同一个位置，或者你想把旧数据发给自己另一台电脑，可以使用迁移包。

在旧 PA 目录里双击：

```bat
数据迁移-打包.bat
```

它会在旧目录根目录生成固定文件名：

```text
PA用户数据迁移包.zip
```

然后把这个 zip 复制到新 PA 目录根目录，再在新目录里双击：

```bat
数据迁移-解压.bat
```

解压脚本会先校验压缩包结构，防止危险路径；然后把配置、人设、聊天状态、日记、定时任务、图书馆、表情包、附件、生成图、AP 自动调参配置、`hdb/data` 等用户数据还原到新目录。还原前会把新目录里已有的同名文件备份到：

```text
observatory/outputs/migration_backups/
```

默认不会打包 `.jsonl` / `.log` / `.tmp` / `.gz` 等运行日志，避免把旧版膨胀日志带到新目录。

## 页面还是旧版怎么办

1. 先确认只启动了一个 PA 后端。可以重新双击 `一键重启进程并更新.bat`，它会先关闭 `8765` 端口旧进程。
2. 确认启动脚本来自你想使用的那个目录，不要再双击另一个旧目录里的启动脚本。
3. 浏览器访问：

```text
http://127.0.0.1:8765/next/
```

4. 如果页面仍然像旧版，按 `Ctrl+F5` 强制刷新浏览器缓存。
5. 确认配置、人设、日记、图书馆和聊天记录都正常后，再把不用的旧目录重命名或移动到备份位置。

## 注意

- 不要手动复制整个源码目录互相覆盖，容易把旧前端、旧脚本和新配置混在一起。
- 不要删除旧目录，直到确认新版里配置和历史数据都正常。
- 如果旧版 `observatory/outputs/` 已经很大，可以在旧目录中运行 `清理日志和临时输出.bat`，先预览，再清理日志。
