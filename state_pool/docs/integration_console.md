# AP 状态池增强联动控制台说明

入口脚本：

- [state_pool/tests/run_integration_enhanced.py](/H:/AP原型测试/state_pool/tests/run_integration_enhanced.py)
- 兼容入口 [state_pool/tests/run_integration.py](/H:/AP原型测试/state_pool/tests/run_integration.py)

## 当前用途
这个控制台不是只看 Top-N，而是用来手工联调当前“文本感受器 -> 状态池”链路是否按预期工作。重点检查：

- `stimulus_packet` 是否包含正确的时序组
- 残响是否被正确并入包体
- 同包内语义同一对象是否先被聚合
- 状态池是否按配置进行中和、对齐、合并与淘汰
- 属性绑定和定向能量更新是否能即时回写

## 当前常用命令
- 直接输入文本：执行“文本感受器 -> 状态池”的一轮联调
- `tick [n]`：执行 1 到 n 轮状态池维护
- `autotick on|off|ask`：控制处理下一条文本前是否自动先做维护
- `packet groups|trails|full`：查看最近一个刺激包的分组、跨组重现和完整内容
- `snap [k]`：查看状态池快照 Top-K
- `inspect [rank|item_id]`：查看单个状态项详情
- `history [n]`：查看最近 n 条状态变化事件
- `bind [rank|item_id] [attribute]`：给对象绑定属性节点
- `energy [rank|item_id] [delta_er] [delta_ev]`：做定向能量更新
- `insert [kind] [label] [er] [ev]`：插入运行时节点
- `attention [k]`：生成注意力占位快照
- `reload`：热加载状态池配置
- `config`：查看关键配置
- `clear`：清空状态池

## 当前观察重点
### 1. 刺激包分组
当前控制台重点看的是组级显示，而不是扁平 token 串。目标是回答：

- 本轮到底有几个时序组
- 哪些是当前输入，哪些是残响
- 同组内部包含哪些特征 SA 和属性 SA

### 2. 残响与当前输入的关系
当前实现中，残响会显式参与当前包体，因此控制台会区分：

- 当前组
- Echo 组
- 跨组重复出现的对象

### 3. 缓存中和
如果启用了优先刺激中和，控制台会显示：

- 哪些目标被尝试中和
- 本轮真正消耗了多少 ER / EV
- 如果中和失败或未发生，还缺多少可中和能量

### 4. 语义同一对象聚合
当前状态池默认会对同包内语义相同的对象先聚合，再与池中已有对象对齐。这一点在联调控制台中非常重要，因为它直接决定：

- 是否会出现无意义的重复状态项
- Echo 与当前输入的能量是否会被错误重复计入

## 当前建议用法
如果你在排查“文本感受器输出和状态池落地是否一致”，推荐顺序是：

1. 输入一条短文本
2. 看 `packet groups`
3. 看 `snap`
4. 再输入第二条文本
5. 看 Echo 是否进入当前包、聚合是否正确
6. 用 `tick` 看衰减和淘汰是否符合预期

这个控制台主要服务于前两模块联调；更完整的结构级、刺激级和赋能链路，请转到观测台或 HDB 综合联动控制台。
