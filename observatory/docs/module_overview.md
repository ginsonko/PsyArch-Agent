# AP 观测台模块总览

## 模块定位
`observatory` 是当前原型阶段的统一观测与联调入口。它不替代 `text_sensor`、`state_pool`、`hdb` 的核心计算，而是把它们串成一个可以直接运行、直接观察、直接改配置并热加载的实验界面。

当前观测台已经覆盖：

- 文本输入与循环触发
- 状态池维护
- 注意力占位与记忆体抽取
- 结构级查存一体
- 缓存中和
- 刺激级查存一体
- 投影回写
- 感应赋能
- 记忆赋能池与记忆反哺
- 配置查看、编辑、保存、热加载

## 当前前端交互原则
### 1. 不再自动定时刷新
当前 Web 前端已经按新的规则工作：

- 不使用定时轮询自动刷新
- 页面复制、选中文本、编辑设置时不会再被周期性刷新打断
- 使用悬浮刷新按钮手动刷新
- 点击页面动作按钮后，会在动作完成后主动刷新一次数据

### 2. 设置页保留草稿
设置页当前按“草稿优先”原则工作：

- 用户未保存的输入不会被刷新覆盖
- 保存后调用对应模块的热加载接口
- 注释直接来自各模块 YAML 配置文件
- 文件值、运行时覆盖值、最终生效值会一并展示

### 3. 统一展示样式
当前观测台统一使用以下显示规范：

- 时序信息组：`{...}`
- 同一 CSA 或同组成员：`A + B`
- 不同时序组之间：`{...} / {...}`

这套显示规范会贯穿：

- 输入包显示
- 刺激级查存一体
- 结构级查存一体
- 残差信息
- 共同结构
- 记忆相关卡片

## 当前主循环顺序
观测台现在对外展示的顺序与实际运行顺序一致：

1. 状态池维护
2. 记忆体抽取
3. 结构级查存一体
4. 缓存中和
5. 刺激级查存一体
6. 状态池回写与结构投影
7. 感应赋能
8. 记忆赋能池与记忆反哺

其中：

- `text_sensor` 负责形成带时序组的 `stimulus_packet`
- `state_pool` 负责运行时对象能量维护与中和
- `hdb` 负责查存一体、局部数据库、记忆赋能与记忆反哺
- `observatory` 负责把这些结果组织成可读报告与设置界面

## 设置模块
### 当前能力
设置页现在已经可以：

- 查看四个模块全部当前配置项
- 读取并显示每个参数的中英双语注释
- 修改参数值
- 保存到对应 YAML 文件
- 调用模块自身 `reload_config()` 接口立即生效

### 注释来源
设置页不是自己维护一套说明文案，而是直接读取：

- `text_sensor/config/text_sensor_config.yaml`
- `state_pool/config/state_pool_config.yaml`
- `hdb/config/hdb_config.yaml`
- `observatory/config/observatory_config.yaml`

因此当前设置界面看到的说明，就是代码真实配置文件里的最新版说明。

## 当前重点观测区域
### 1. 文本感受器
显示：

- SA
- CSA
- 时序组
- Echo/残响合并情况

### 2. 结构级查存一体
当前重点展示：

- 进入结构级的结构
- 每轮预算前后
- 选中结构组
- 链式打开路径
- 局部库动作
- 原始残差与还原结构
- 新建结构组

### 3. 缓存中和
当前重点展示：

- 输入包与剩余包
- 结构级实时生效后，完整刺激可中和的部分
- 若无可中和 EV，也会保留“所需量 / 缺口”语义，而不是简单误导为没有差额

### 4. 刺激级查存一体
当前重点展示：

- 每轮 Anchor
- 局部工作组
- 轮前残余与轮后残余
- 候选结构与命中结构
- 新建共同结构
- 新建残差结构
- 结构路径扩展

### 5. 记忆赋能池
当前重点展示：

- 当前记忆池 EV
- 最新激活事件
- 反馈次数
- 最近反馈与累计反馈
- 结构引用、结构组引用
- 排序方式

当前排序支持按：

- 能量从高到低优先
- 同能量时按时间从新到旧

## 当前对外接口
运行与控制接口：

- `/api/dashboard`
- `/api/cycle`
- `/api/tick`
- `/api/reload_config`
- `/api/clear`
- `/api/self_check`
- `/api/repair`

查询接口：

- `/api/structure`
- `/api/group`
- `/api/episodic`

这些接口共同支撑 Web 页面、CLI 输出和 HTML/JSON 导出。

## 当前阶段总结
观测台当前已经不只是一个“看快照”的页面，而是：

- 真实反映当前代码流程
- 能直接改配置并立即生效
- 能稳定观察结构级、刺激级、记忆池与反哺链路
- 能为后续时间感受器模块继续提供可读日志和调试入口

当前它最重要的价值是把复杂理论过程压成“能看见、能对照、能验收”的实验界面。
