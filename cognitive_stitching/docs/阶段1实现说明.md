# 认知拼接模块阶段 1 实现说明

本阶段只落地一个保守、可回滚、可验收的最小版本，目标不是一次完成全部理论细节，而是先把主链插入点、开关、安全性和基础观测打稳。

## 已落地范围

- 插入位置固定为：`StatePool maintenance -> Cognitive Stitching -> Attention`
- 使用 `HDB diff_table` 作为候选边来源，不做全局扫描
- 只实现最小的双结构拼接，不做事件组扩展、懒融合、组分账本
- 拼接后的事件节点以 `ST-compatible runtime node` 进入状态池，能够被注意力看到
- 同一对拼接具备数值疲劳，避免短时间反复抽能过强
- 输出基础叙事化报告：
  - 最近拼接动作
  - 当前拼接候选摘要
  - 当前状态池中能量靠前的 CS 事件节点

## 本阶段有意暂缓的内容

- 事件组到事件组的多级扩展与上下文回退
- ES 独立数据库、懒融合、闲时巩固
- ES 内部组分能量账本与缓存中和联动
- 刺激级全链路弱命中推广
- 历史 SA 粒度能量图景存储

其中“时序弱命中向刺激级全链路推广”和“历史能量图景入库”已单独冷保存，后续以开关实验方式评估，不在当前阶段直接改动主逻辑。
