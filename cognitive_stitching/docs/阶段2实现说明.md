# 认知拼接模块阶段 2 实现说明

本阶段在阶段 1 的保守双结构拼接基础上，继续向设计文档靠近，但仍坚持“先可控、再扩展”的落地原则。

## 本阶段已落地能力

- 保留阶段 1 的 `structure -> structure` 事件创建能力。
- 允许已有的 CS 事件节点重新进入候选池，作为新的拼接种子。
- 新增保守的右端扩展：
  - 形式为 `[A,B] + C -> [A,B,C]`
  - 当前通过事件尾部组分的局部 `diff_table` 残差边驱动
  - 仍然只做端点扩展，不做中间插入
- 新增保守的事件到事件桥接：
  - 形式为 `[A,B,C,D] + [E,F,G,H] -> [A,B,C,D,E,F,G,H]`
  - 当前要求左事件尾部组分能通过局部残差边命中右事件头部
  - 支持“右事件头部前缀”命中，因此 `D -> E`、`D -> E+F`、`D -> E+F+G` 都可以形成不同强度的桥接
- 新增保守版 ES 组分账本与缓存中和联动：
  - CS 事件在写入状态池时保留 `member_refs`、`component_profile` 与 `component_ledger`
  - 优先中和阶段允许对命中的事件组分做“互补补足”，只补更小的一侧
  - 同向能量不会被当作中和能量塞回事件整体，因此更符合“中和降压、同向堆叠促碎片化”的设计原则
- 保留并继续强化数值化软疲劳：
  - 同一路径重复抽能会越来越弱
  - 但不会被硬编码完全禁止
- 审计字段补强：
  - `action_family`
  - `context_k`
  - `matched_span`
  - `source_kind / target_kind`
  - `seed_plain_structure_count / seed_event_count`
  - `created_count / extended_count / merged_count / reinforced_count`

## 本阶段刻意保持保守的地方

- 仍未引入真正的 `parents + delta` 懒融合 ESDB。
- 已引入运行态保守版 ES 组分账本与缓存中和联动，但仍未扩展到懒融合 ESDB、闲时巩固与更复杂弱命中。
- 仍未实现左端前置与全量 suffix/prefix pointer index 回退搜索。
- 仍未实现“事件结构进入 HDB 并拥有独立数据库”的正式持久化。
- 仍未把时序弱命中推广到刺激级查存一体主流程。
- 仍未引入历史 SA 粒度能量图景模板。

## 当前实现的核心折中

本阶段没有直接实现设计文档中的完整“上下文结构数据库回退”，而是采用了一个更稳妥的近似方案：

- 事件扩展或事件桥接时，只打开事件尾部最近 `k` 个组分各自的局部 `diff_table`
- 若多个尾部组分都能支持同一目标，则 `context_k` 更高、得分更高
- 若只有更靠前的尾部组分能支持目标，则允许命中，但会因 `closest_distance` 受到惩罚

这样做的好处是：

- 不需要新建 HDB 指针索引或复合上下文数据库
- 仍然能表达“D 支持最强，C+D 联合支持更强，只有 C 支持则更弱”的连续谱
- 与现有 HDB `diff_table` 契约兼容，不会破坏刺激级与结构级现有逻辑

## 当前验收覆盖

- 单元测试覆盖：
  - 阶段 1 的基础双结构创建
  - 已有事件的右端扩展
  - 事件到事件桥接后的更长事件生成
- 主链集成测试覆盖：
  - 观测台运行一轮后能产出 CS 报告
  - 已有 CS 事件种子可在主链中被继续扩展
- 观测输出覆盖：
  - 终端报告可看到扩展/桥接相关字段
  - HTML 报告的 CS 表格已增加动作族、上下文长度和命中跨度字段

## 下一阶段建议

若继续推进，建议顺序如下：

1. 在现有保守版 ES 组分账本与缓存中和联动基础上，继续补强更复杂弱命中与持久化账本。
2. 再补更严格的 suffix/prefix 回退式桥接。
3. 再评估 ESDB 懒融合与闲时巩固。
4. 最后再考虑事件把握感、APT 更细粒度规则，以及更强的前端叙事化展示。
