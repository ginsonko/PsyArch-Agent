PsyArch Agent BAT 脚本使用说明
================================

这份说明面向不熟悉命令行的 Windows 用户。所有 bat 文件一般都可以双击运行。
如果脚本提示缺少 Python / Git / PowerShell / Node.js，请先安装对应环境。

一、第一次安装与启动
------------------

1. 自动拉取最新版PA.bat
   用途：给还没有 PA 项目的新用户使用，一键从 GitHub 拉取或更新 PsyArch-Agent。
   放置位置：建议放在一个空的上级目录中，例如 E:\PA\自动拉取最新版PA.bat。
   运行结果：会在同级目录生成或更新 PsyArch-Agent 文件夹。
   注意：它是“拉取到脚本所在目录下的 PsyArch-Agent 文件夹”，不是更新任意旧目录。

2. 依赖自检与安装.bat
   用途：第一次使用 PA 时安装 Python 依赖，创建 .venv。
   什么时候用：首次安装、依赖损坏、换电脑、更新后提示缺包时。
   放置位置：PsyArch-Agent 项目根目录内，保持默认即可。

3. 快速启动观测台.bat
   用途：启动 PA 后端和观测台。
   什么时候用：日常本地启动 PA。
   启动后访问：http://127.0.0.1:8765/next/
   注意：如果已经有旧进程占着 8765，页面可能仍然是旧版；这种情况优先用“一键重启进程并更新.bat”。

4. 一键启动PA-Agent与AP后端.bat
   用途：新版推荐启动入口之一，会用固定端口 8765 启动 PA-Agent + AP 后端，并打开浏览器。
   什么时候用：希望直接启动完整 Agent 页面时。
   和“快速启动观测台.bat”的关系：两者都能启动，普通用户用这个或快速启动都可以。

二、更新、旧进程、页面还是旧版
----------------------------

5. 一键重启进程并更新.bat
   用途：解决“更新了但页面还是旧版”“8765 旧进程关不掉”“用户本地有 ZIP 版旧目录”的问题。
   什么时候用：最推荐的日常更新方式，尤其是已经配置好 API Key、人设、QQ 白名单、日记、图书馆、HDB 数据后。
   使用方式：把这个 bat 放在你正在使用、想要原地更新的 PA 文件夹根目录里，双击运行。
   它会做什么：
   - 关闭占用 8765 端口的旧 PA 进程；
   - 如果当前目录是 Git 仓库，就拉取 main 分支最新版；
   - 如果当前目录是 GitHub ZIP 解压目录、没有 .git，就临时下载最新版源码并覆盖当前目录；
   - 保留 observatory/outputs、hdb/data、.venv、node_modules、NapCat、日志等本地数据；
   - 无论更新成功或失败，都会重新启动 PA。
   注意：它更新的是“bat 所在的当前目录”，不会去外面新建第二个 PsyArch-Agent。

三、旧版数据迁移
----------------

如果本地同时有 PsyArch-Agent 和 PsyArch-Agent-main：
- PsyArch-Agent-main 通常是 GitHub ZIP 解压出来的旧目录；
- PsyArch-Agent 通常是一键拉取脚本克隆出来的新目录；
- 两个目录的数据互不相通。

优先建议：如果旧目录里已经有配置和历史数据，就把“一键重启进程并更新.bat”放进旧目录原地更新。
只有你确定以后想切换到新目录时，才需要迁移数据。

6. 数据迁移-打包.bat
   用途：在旧 PA 目录里，把用户数据打包成固定名称的压缩包。
   使用方式：在旧目录里双击运行。
   生成文件：PA用户数据迁移包.zip
   默认打包内容：
   - observatory/outputs/agent 下的配置、人设、对话状态、日记、定时任务、图书馆、表情包、附件、生成图、运行包等；
   - observatory/outputs/auto_tuner 下的 AP 自动调参配置和持久化覆盖；
   - hdb/data 下的 AP/HDB 长期运行数据。
   默认不会打包：.jsonl、.log、.tmp、.gz 等容易膨胀的运行日志。

7. 数据迁移-解压.bat
   用途：在新 PA 目录里，还原“PA用户数据迁移包.zip”。
   使用方式：先把旧目录生成的 PA用户数据迁移包.zip 复制到新目录根目录，再在新目录双击运行。
   安全机制：
   - 解压前会校验压缩包路径，拒绝 ../escape.txt 这类越界路径；
   - 覆盖同名文件前，会备份到 observatory/outputs/migration_backups/。
   还原后：从新目录启动 PA，并按 Ctrl+F5 强制刷新浏览器页面。

8. 一键迁移旧版PA用户数据.bat
   用途：不用手动复制 zip，直接从旧目录复制用户数据到当前新目录。
   使用方式：在新目录里运行。脚本会自动找同级 PsyArch-Agent-main，也可以手动粘贴旧目录路径。
   适用场景：旧目录和新目录在同一个上级目录，比如：
   - E:\PsyArch-Agent
   - E:\PsyArch-Agent-main
   注意：目标目录已有同名数据会先备份；旧目录不会被删除。

四、NapCat 相关
---------------

9. 一键拉取或更新NapCat.bat
   用途：拉取或更新 NapCatQQ 源码。
   什么时候用：用户需要 QQ 对接，但本地没有 NapCat，或想更新 NapCat。
   默认目录：PA 仓库同级的 NapCatQQ 文件夹。

10. 一键配置NapCat连接PA.bat
    用途：自动给本地 NapCat 写入 PA 默认 webhook 配置。
    什么时候用：已经拉好 NapCat 后，希望自动添加 PA 连接。
    默认 PA webhook：http://127.0.0.1:8765/api/agent/napcat/event
    注意：已有 NapCat 的用户也可以按 docs/已有NapCat手动配置.md 手动配置。

11. 一键打开NapCat.bat
    用途：启动同级目录里的 NapCatQQ。
    什么时候用：PA 已经启动后，打开 NapCat 登录 Bot QQ。

12. start-napcat-pa.bat
    用途：旧的兼容启动入口，配合 start-napcat-pa.ps1 使用。
    普通用户一般不需要优先使用它。

五、清理日志和磁盘占用
----------------------

13. 清理日志和临时输出.bat
    用途：清理旧版可能产生的大量日志、临时探针输出和历史报告，避免磁盘继续膨胀。
    什么时候用：observatory/outputs 占用很大，或用户从旧版升级后想释放空间。
    推荐步骤：先选择 Preview only 预览，再选择保留 1 天或清理全部目标。
    会保留：配置、HDB 数据、latest.json/latest.html、表情包、生成图片、收到的附件。
    会清理或压缩：历史 cycle 报告、临时输出、过大的 jsonl/log 等。

六、常见问题
------------

Q1：我更新后页面还是旧版怎么办？
A：运行“一键重启进程并更新.bat”，它会先关闭 8765 旧进程。然后打开 http://127.0.0.1:8765/next/ 并按 Ctrl+F5。

Q2：为什么我有 PsyArch-Agent 和 PsyArch-Agent-main 两个目录？
A：一个通常是 Git 克隆版，一个是 GitHub ZIP 解压版。它们互不共享配置。推荐在已有配置的目录里原地更新，或用数据迁移脚本迁移。

Q3：迁移会不会把我的 API Key 和 QQ 配置弄丢？
A：迁移脚本会复制 agent_config 等用户配置；解压前还会备份新目录已有同名文件。不要删除旧目录，直到确认新目录正常。

Q4：迁移包会不会很大？
A：默认跳过 .jsonl/.log/.tmp/.gz 等日志，所以不会把旧版膨胀日志一起打包。但如果图书馆、附件、生成图很多，迁移包仍可能变大。

Q5：只把 bat 发给旧目录用户可以吗？
A：可以。迁移和更新相关 bat 如果发现 tools 里的 helper 缺失，会尝试从 GitHub 最新仓库临时拉取 helper 后继续执行。

建议新手最常用顺序
------------------

第一次安装：
1. 自动拉取最新版PA.bat
2. 进入 PsyArch-Agent
3. 依赖自检与安装.bat
4. 一键启动PA-Agent与AP后端.bat
5. 打开页面填写 API Key 并保存

日常更新：
1. 在正在使用的 PA 目录里运行 一键重启进程并更新.bat

旧目录切换到新目录：
1. 旧目录运行 数据迁移-打包.bat
2. 把 PA用户数据迁移包.zip 复制到新目录
3. 新目录运行 数据迁移-解压.bat
4. 新目录运行 一键重启进程并更新.bat
