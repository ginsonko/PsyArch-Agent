# AP HDB 模块总览

## 模块定位
`hdb` 是当前原型里的长期结构化记忆层。它负责把运行中形成的结构、结构组、残差信息和情景记忆稳定地存下来，并在后续循环里按局部数据库优先的方式重新打开、匹配、赋能和回写。

HDB 当前已经打通的主链路包括：

- 结构级查存一体
- 刺激级查存一体
- 感应赋能
- 独立记忆赋能池
- 记忆赋能反哺到状态池原料

## 当前实现边界
当前 HDB 已经明确区分三类对象：

- `structure / st / sa / csa`：可参与结构级、刺激级和感应赋能主循环
- `group / sg`：结构级组合结果，用于结构级局部数据库继续积累
- `episodic memory / em`：情景记忆信息，只进记忆池，不作为运行时结构对象进入状态池

这里最重要的约束是：

- 记忆信息不是结构信息
- 记忆不会嵌套生成新的记忆
- 记忆不会进入状态池成为 `ref_object_type == "em"` 的运行时项
- 记忆被激活后，只会把能量按颗粒分配规则“反哺”给其组成材料

## 当前主流程
### 1. 结构级查存一体
结构级流程以 `ST` 粒度处理 CAM/注意力占位结果。

当前已经实现：

- 结构组竞争
- 结构组局部库链式打开
- 残差记录写入结构数据库或结构组本地库
- 只有“最大共同部分既包含当前选中结构组、又满足时序对齐”时才创建新的共有结构组
- 数值刺激元在结构级匹配中允许同类模糊匹配，结构判定上可视为匹配，分数上按数值差距折扣

### 2. 刺激级查存一体
刺激级流程以当前 `stimulus_packet` 为输入，在“当前工作组”局部范围内做贪婪链式匹配，不做全局扫描。

当前已经实现：

- 刺激包保留时序信息组
- 同一时序组内顺序不敏感，不同组之间顺序敏感
- 残差结构和共同结构都保留时序组信息
- 统一展示样式为：
  - 时序信息组：`{...}`
  - 同组成员：`A + B`
  - 跨组顺序：`{...} / {...}`
- 只有在“当前选中结构被包含，且最大共同部分的时序结构正确”时，才允许创建新的共同结构
- 当残差完全相同，只强化已有记录，不重复造新共同结构

### 3. 数值刺激元
数值刺激元当前分成两个层面处理：

- 打开局部数据库时：使用数值桶（numeric buckets）
- 真正做匹配、切割、共同部分计算时：使用同类数值元模糊匹配

这两层逻辑不冲突：

- 桶只负责“找哪几个本地库”
- 匹配只负责“结构上算不算匹配、分数打多少折”

当前设计包括：

- 最近邻数值桶联合打开，等效为该数值元的第一跳本地库
- 同类数值元允许模糊匹配
- 数值相似系数采用小值除以大值
- 共同结构中若涉及数值共同部分，取平均值而不是桶中心
- 数值桶支持受限数量、自适应分裂、必要时合并

## CSA 当前语义
当前代码与文档统一采用以下语义：

- CSA 是若干 SA 的组合表达，不是单独拥有能量的运行时对象
- 状态池里的 ER/EV 跟随 SA/ST 走，不跟随 CSA 独立存在
- CSA 在查存一体里主要承担“组合约束”和“完整包含关系判断”
- 如果刺激侧 CSA 不能完整包含结构侧 CSA 需要的成员，则视为不能完全匹配

## 记忆赋能池与记忆反哺
### 独立记忆赋能池
当前已经实现独立记忆赋能池，用于接收两类来源的 EV：

- ER 诱发到记忆
- EV 传播到记忆

前端可直接查看：

- 当前记忆池 EV
- 最近激活事件
- 最近反哺事件
- 总反哺 EV

### 记忆反哺
当前已经实现“记忆池保留独立性，同时把记忆内容按颗粒反哺回状态池材料”的链路。

核心规则：

- 不把记忆对象本身放进状态池
- 只把记忆内容拆成最小颗粒后按初始能量分布比例回写
- 结构级记忆拆成结构颗粒
- 刺激级记忆拆成 SA/CSA 对应的最小材料颗粒

这样做的结果是：

- 记忆可以重新参与后续认知循环
- 但不会出现“记忆嵌套记忆”的无限递归

## 局部数据库原则
HDB 当前坚持以下性能原则：

- 运行时查询优先走 `owner -> db pointer -> local db`
- 只有在受限条件下才允许 fallback
- `allow_global_scan_on_runtime_path` 默认关闭
- 数值刺激元第一跳也走“最近桶本地库”，不是全库搜索

这保证当前主路径与数据库规模弱相关，而不是随着总库规模线性恶化。

## 重要配置项
HDB 配置文件已经为全部当前参数补充中英双语说明，且设置页可直接读取这些注释。

当前最关键的一组参数是近因增益：

- `recency_gain_boost`
- `recency_gain_peak`
- `recency_gain_hold_rounds`
- `recency_gain_refresh_floor`
- `recency_gain_decay_mode`
- `recency_half_life_ms`
- `recency_gain_decay_ratio`

如果你要实现“最高 10 倍，约 100 万 tick 逐步衰减回 1 倍”的效果，当前推荐配置为：

- `recency_gain_peak: 10.0`
- `recency_gain_hold_rounds: 2`
- `recency_gain_refresh_floor: 0.45`
- `recency_gain_decay_mode: by_round`
- `recency_gain_decay_ratio: 0.9999976974`

## 当前阶段总结
当前 HDB 不是理论终版，但已经是一个可运行、可观测、可热更新、可做局部修复的原型记忆骨架。

它现在最重要的价值不是“记得很多”，而是：

- 结构逻辑已经基本稳定
- 局部链式路径清楚
- 时序组与 CSA 语义清楚
- 记忆与结构的边界清楚
- 后续时间感受器模块已经有可接入的记忆池与日志基础
