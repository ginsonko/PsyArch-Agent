# AP HDB 综合联动控制台说明

控制台脚本：

- [hdb/tests/run_full_integration.py](/H:/AP原型测试/hdb/tests/run_full_integration.py)

## 当前用途
这个脚本用于把当前已经打通的主循环串起来做完整联调。它关注的不只是 HDB 自身，而是：

1. 文本感受器输出
2. 状态池接收与维护
3. 注意力占位结果
4. 结构级查存一体
5. 缓存中和
6. 刺激级查存一体
7. 状态池回写
8. 感应赋能
9. 记忆赋能池
10. HDB 快照、自检与修复状态

## 当前常用命令
- `text <文本>`：输入文本并立即执行一轮完整流程
- `tick [n]`：在无新输入时执行 n 轮循环
- `snap [k]`：查看状态池快照
- `hdb`：查看 HDB 摘要
- `st <structure_id>`：查看单个结构与其局部数据库
- `sg <group_id>`：查看单个结构组
- `em [n]`：查看最近情景记忆
- `check [target]`：运行 HDB 自检
- `repair <target>`：执行局部修复
- `repair_all`：启动全局快速修复
- `stop_repair <job_id>`：停止后台修复
- `clear_hdb`：清空 HDB
- `clear_all`：清空 TextSensor echo、StatePool、HDB
- `config`：查看关键配置
- `reload`：热加载三模块配置
- `help`
- `quit` / `exit`

## 当前最值得观察的内容
### 1. 刺激级日志
重点观察：

- 当前局部工作组是什么
- 命中了哪个结构
- 新建了什么共同结构
- 新建了什么残差信息
- 剩余刺激在每轮之后还剩多少

### 2. 结构级日志
重点观察：

- 当前选中的结构组或单结构锚点是什么
- 命中了哪个局部数据库
- 追加了什么原始残差信息
- 还原结构是什么
- 对应 `em_id` 是什么

### 3. 记忆赋能池
当前综合控制台和观测台都已经能看到：

- 哪些记忆体被赋能
- 当前 EV 多高
- 来自哪些结构的传播或诱发
- 最近更新时间

## 当前调试建议
如果你要检查“某个现象到底是结构级有问题，还是刺激级有问题”，推荐顺序是：

1. 先看当前输入在 TextSensor 里分成了哪些组
2. 再看结构级有没有产生不合理的结构组残差信息
3. 再看刺激级的局部工作组是否正确
4. 最后看感应赋能和记忆赋能池是否跟着命中的结构/记忆走

## 当前与观测台的关系
这个脚本适合终端联调和快速回归；观测台适合更直观地看：

- 时序组
- CSA 显示
- 字段级配置说明
- 前端检视器
- 记忆赋能池排序

两者展示重点不同，但底层主流程是一致的。
