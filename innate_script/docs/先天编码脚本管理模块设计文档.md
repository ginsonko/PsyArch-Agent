# 先天编码脚本管理模块（IESM）设计文档（模块 8）

生成时间：2026-04-12  
理论依据：`txt版本的理论核心.txt`（重点参考：3.10、3.8、3.9、4.2.1、3.12）

> IESM（Innate Encoded Script Manager）是 AP 的“人工 DNA”：把关键规则集中管理，使系统具备可控的先天倾向与安全边界，同时让调制与触发逻辑可版本化、可回滚、可审计。

---

## 1. 模块定位与职责边界

### 1.1 理论定位（3.10）
IESM 管理三类脚本：
1. **认知感受脚本（CFS 脚本）**
   - 定义如何从 `(E_r,E_v,P)`、匹配质量、残差、赋能稳定性、奖惩共现等生成 CFS
   - 定义融合/抑制逻辑与“正确事件”触发条件
2. **情绪脚本（EMgr/NT 脚本）**
   - 定义 NT 通道更新/衰减/上下限
   - 定义 NT/Rwd/Pun 对 AF/HDB/Drive/行动阈值的映射
3. **先天行动触发规则脚本（Innate Triggers）**
   - 定义当满足条件时如何带参/不带参地触发行动器对应的行动节点
   - 触发结果是“对行动节点赋能/赋 Drive”，不是强制立即执行

### 1.2 工程职责
- 脚本加载、校验、版本管理、热加载
- 规则执行（输入：状态窗口/快照；输出：触发结果与调制参数）
- 冲突仲裁（优先级、安全底线）
- 审计记录（脚本版本、触发链、输入摘要、输出摘要）

## 2. “每个行动器至少一条先天规则”原则（3.10.2）

理论强调：让系统在冷启动阶段就“知道自己会做这个行动”，并把行动节点纳入可学习闭环。

### 2.1 注意力行动器（带参聚焦）示例（理论示例 1）
- 先天规则：当 SP 中出现违和感/正确事件/期待/压力等 CFS 对象时，将该 CFS 绑定的 CSA 作为参数，带参触发“注意力聚焦”行动器
- 参数含义：聚焦目标为该 CSA 的锚点特征与关键属性（如“某结构 + 违和感:高”）
- 执行效果：AF 在下一 tick 对“完全匹配/高度相似”的对象给予额外增益

这直接决定模块间接口形式：
- IESM 输出 `focus_directives`（包含 CSA 参数）
- AF 接收并在下一 tick 生效

## 3. 工程化要求（3.10.3）

IESM 建议具备：
- 脚本版本号
- 变更记录
- 可回滚机制
- 权限与沙箱
- 规则冲突优先级仲裁（通常安全类/终止类最高）

这些能力是“长期可控、可解释”的前提，建议从原型阶段就预留字段与文件结构。

---

## 4. 输入输出接口（原型建议）

### 4.1 输入：状态变化窗口与关键快照
状态池模块当前已经具备“脚本检查抄送包”能力（见 state_pool 的 broadcast_script_check 逻辑）。
IESM MVP 可以先接收：
- `script_check_packet`（包含候选触发线索、对象变化摘要）
- `state_snapshot`/`cam_snapshot`（可选，用于更复杂规则）

### 4.2 输出：三类结果
1. `triggered_scripts`：触发的脚本列表（含优先级、参数、解释）
2. `directives`：给其他模块的带参指令
   - `focus_directives`（给 AF）
   - `emotion_updates`（给 EMgr）
   - `action_triggers`（给 Action/Drive）
3. `audit_log`：用于观测台展示与离线审计

---

## 5. 现阶段接入策略

由于模块 5-7 尚未实现，IESM MVP 建议先做“只读 + 触发结果可观测”：
- 能接收 state_pool 的 script_check_packet
- 能输出 triggered_scripts（为空也要结构化返回）
- 能在 observatory 中显示脚本版本与触发统计

当模块 5（CFS）完成后：
- IESM 将把 CFS 生成规则外置脚本化

当模块 6（EMgr）完成后：
- IESM 将把 NT 更新与调制映射脚本化

当模块 7（Action/Drive）完成后：
- IESM 将把“先天行动触发”脚本化，并真正驱动注意力聚焦/回忆行动/语言行动等内外行动器

## 6. 原型落地补充（2026-04-12 之后）

为降低联调成本，原型阶段已新增“声明式规则文件 + 观测台前端编辑”：

- 规则文件：`innate_script/config/innate_rules.yaml`
- 备份目录：`innate_script/config/rules_history/`
- 观测台入口：侧边栏「先天规则 / Innate Rules」
- 使用说明：见 `innate_script/docs/先天规则编辑指南.md`
